All content is my own work, no ai generated content or external assets used. All rights reserved. i did install an addon to integrate aseprite compatability, but i did not end up using it due to being unfamiliar with how it works.

this project is one of my favourite creations so far learning game design. i was able to really push my abilities in the godot engine, and learned a lot of more complex features and how they work. I explored new areas of what the engine can do, from deffered loading to material properties and sprite billboarding.

the initial idea was to just have a 2d sidescroller of the bus and luas journey to and from college that i experience every day. this idea limited me somewhat, and i found that i was not able to incorporate the luas in a way that i felt would meet the standards i kept for the rest of the game. i then decided to have a 3d platformer section, heavily inspired by hit game Only Up. the 3d platforming section is definitely my favourite, as i felt that recreating the university building in a game was a very fun undertaking.

i created every asset added into the project, creating them in aseprite and then importig them as pngs. i prefer to work in low poly and low res art styles, heavily inspired by mario 64, a childhood favourite of mine. 

i used git and github as version control throughout the project, and found it very useful for storing my project across my laptop and pc, letting me work at home and out and about.

the system i am most proud of is the deffered loading system. while loading on the main thread stops all processes of the game while it loads, a deffered loading system lets me have a loading screen with a functional progress bar, and even load scenes during gameplay. this is very useful as it lets me avoid long load times, which is esspecially annoying on low end systems.

another feature i quite enjoyed making is the settings system. it uses a global class that lets every scene access the settings , making the implementation of various settings possible with ease. the 3 avialable settings are view sensitivity, a speedrun timer to test your skill, and the option to not normalise movement on diagonals, a call back to the classic DOOM series of games.

overall this project was really fun, and was made even better by being made in my favourite engine. it has been great fun to work on, and the end result feels very good.
